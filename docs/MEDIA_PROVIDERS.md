# Реальные картинки DAS: WIS и StarFinder

В библиотеке Vediamo схем проводки и расположения компонентов **нет** — они в
WIS/Xentry и StarFinder. macDiag умеет подключать их как «медиа-провайдеры»: в
панели «провалиться в ошибку» сначала идут наши генерируемые SVG-схемы, а ниже —
реальные картинки из подключённого датасета.

## Что уже работает без этих данных

- Drill-down по ошибке: вероятные причины, шаги проверки, связанные группы
  измерений и процедуры, генерируемая схема CAN-шины (и расположения цилиндров
  для моторных ЭБУ).

## StarFinder в формате .asar (как на скриншоте)

StarFinder хранит данные по сериям кузова в Electron-архивах: `resources/164.asar`
(твой X164/GL), `221.asar` (S-class) и т.д. Этот формат macDiag читает напрямую —
распаковывать не нужно. `backend/mb/asar.py` достаёт нужный файл/картинку из архива
по смещению, открывая только заголовок + один файл (быстро даже для 230 МБ).

Просто укажи папку `resources`:

```bash
export MACDIAG_STARFINDER_DIR="/путь/к/StarFinder/resources"
```

`/api/diag/providers` тогда покажет найденные архивы (`archives: ["164.asar", …]`).

Чтобы я написал привязку «ошибка/ЭБУ → нужная страница и картинка», осмотри свой
архив и пришли вывод:

```bash
python3 tools/inspect_asar.py /путь/к/164.asar            # дерево + расширения
python3 tools/inspect_asar.py /путь/к/164.asar --cat <страница.htm>   # пример страницы
```

## Как подключить StarFinder (распакованный HTML)

1. Перенеси/скопируй экспорт StarFinder на Mac (или примонтируй диск с Windows и
   укажи путь к нему). Нужен корень, где лежат `.htm/.html` страницы и картинки.
2. Укажи путь приложению и перезапусти backend:

   ```bash
   export MACDIAG_STARFINDER_DIR="/путь/к/StarFinder"
   # перезапуск uvicorn backend.main:app
   ```

3. Проверь, что провайдер виден: открой `/api/diag/providers` — у `starfinder`
   должно стать `configured: true`.

## Как подключить WIS/ASRA (тяжелее)

```bash
export MACDIAG_WIS_DIR="/путь/к/WIS"
```

WIS хранит документы/картинки в своей структуре (часто проприетарные форматы или
встроенная БД), поэтому индексатор пишется под конкретный дамп.

## Что мне прислать, чтобы я написал индексатор

Парсер ещё не написан — он зависит от того, как именно разложен твой датасет.
Когда примонтируешь, скинь:

1. **Дерево каталогов** (2–3 уровня) корня StarFinder/WIS:

   ```bash
   find "$MACDIAG_STARFINDER_DIR" -maxdepth 3 | head -100
   ```

2. **Расширения и количество файлов**:

   ```bash
   find "$MACDIAG_STARFINDER_DIR" -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn | head
   ```

3. **Один пример HTML-страницы** с картинкой (например схема проводки какого-то
   узла) — путь к `.htm` и к картинке рядом, чтобы понять, как страница ссылается
   на изображение и как привязать её к ЭБУ/компоненту/коду ошибки.

По этим трём вещам я допишу `backend/mb/media.py:lookup()` так, чтобы он находил
нужные страницы/картинки по ЭБУ и коду ошибки и отдавал их в панель.

## Безопасность

Провайдер отдаёт файлы только из заданного корня (проверка пути в
`resolve_file`), наружу за пределы каталога ничего не уходит.
